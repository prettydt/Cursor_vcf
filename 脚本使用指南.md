## 🚀 快速开始

### 第一步：安装Python（5分钟）

**Windows用户：**
```bash
# 1. 访问 python.org 下载Python 3.8+
# 2. 安装时勾选 "Add Python to PATH"
# 3. 验证安装
python --version
```

**Mac用户：**
```bash
# 使用Homebrew安装
brew install python3

# 验证安装
python3 --version
```

### 第二步：安装依赖（2分钟）

```bash
# 进入项目目录
cd /path/to/Cursor_vcf

# 安装所需库
pip install -r requirements.txt

# 或者单独安装
pip install requests pandas openpyxl pytrends
```

### 第三步：运行脚本（1分钟）

```bash
# 运行脚本
python 需求发现自动化脚本.py

# 按照提示选择模式和关键词
# 等待搜索完成
# 查看生成的Excel文件
```

## 💡 功能说明

### 支持的平台

| 平台 | 状态 | 说明 | 推荐度 |
|------|------|------|--------|
| **V2EX** | ✅ 完全支持 | 有公开API，无需配置 | ⭐⭐⭐⭐⭐ |
| **知乎** | ⚠️ 部分支持 | 需要处理反爬虫 | ⭐⭐⭐ |
| **Product Hunt** | ⚠️ 需要配置 | 需要API token | ⭐⭐⭐⭐ |
| **Google Trends** | ✅ 完全支持 | 需要pytrends库 | ⭐⭐⭐⭐⭐ |

### 三种搜索模式

#### 1. 快速模式（推荐新手）
- **平台**：仅V2EX
- **时间**：2-3分钟
- **优点**：最快，无需配置，稳定可靠
- **缺点**：数据量较少
- **适合**：快速验证想法，了解技术圈需求

#### 2. 标准模式（推荐）
- **平台**：知乎 + V2EX
- **时间**：5-10分钟
- **优点**：数据丰富，覆盖面广
- **缺点**：知乎可能遇到反爬虫
- **适合**：系统性需求发现

#### 3. 完整模式（高级）
- **平台**：知乎 + V2EX + Product Hunt
- **时间**：10-15分钟
- **优点**：数据最全，国内外兼顾
- **缺点**：需要配置API
- **适合**：深度研究，全球视角

### 输出内容

脚本会生成Excel文件，包含以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 平台 | 数据来源 | 知乎、V2EX |
| 标题 | 问题/主题标题 | "有什么好用的PDF转换工具？" |
| 链接 | 原始链接 | https://... |
| 关键词 | 搜索关键词 | "效率工具" |
| 热度 | 关注数/点赞数 | 1523 |
| 回答数 | 回答/评论数 | 45 |
| 发现时间 | 记录时间 | 2024-12-05 10:30:00 |

### 任务3：分析结果（15分钟）

#### 步骤3.1：打开Excel文件（1分钟）
双击 `需求发现结果_XXXXXX.xlsx`

#### 步骤3.2：按热度排序（2分钟）
1. 点击"热度"列标题
2. 点击"数据" → "排序" → "降序"
3. 高热度需求排在前面

#### 步骤3.3：筛选高质量需求（5分钟）

**筛选条件**：
- 热度 > 50（说明有足够关注）
- 回答数 > 10（说明有讨论度）
- 标题清晰（能看懂需求）

**操作**：
1. 在Excel中添加"是否关注"列
2. 对符合条件的打✓
3. 筛选出10-15个待深入分析的需求

#### 步骤3.4：Google Trends验证（5分钟）

对于筛选出的需求：
1. 提取核心关键词（如："PDF转换"）
2. 访问 Google Trends (trends.google.com)
3. 输入关键词查看趋势
4. 记录搜索热度到Excel

**判断标准**：
- 搜索量持续上升：✅ 优先做
- 搜索量稳定：⚠️ 可以考虑
- 搜索量下降：❌ 谨慎考虑

#### 步骤3.5：RICE评分（2分钟）

在Excel中添加RICE评分列：
- Reach（触达）：2-10分
- Impact（影响）：1-3分
- Confidence（信心）：0.5-1.0
- Effort（成本）：1-10分
- RICE = (R × I × C) / E

**示例**：
```
需求：Excel转VCF工具
R: 8 (月搜索5000+)
I: 3 (大幅提升效率)
C: 0.8 (比较确定)
E: 3 (3天能做完)
RICE = (8 × 3 × 0.8) / 3 = 6.4
```

✅ **完成标志**：有1-2个RICE评分>5的高价值需求

## 🔧 进阶配置

### 配置Product Hunt API

#### 步骤1：申请API Token（10分钟）

1. 访问 https://api.producthunt.com/v2/oauth/applications
2. 点击 "Create an Application"
3. 填写信息：
   - Name: Demand Discovery Bot
   - Website URL: https://github.com/yourusername
   - Redirect URI: http://localhost
4. 提交后获得 API Token

#### 步骤2：配置脚本（2分钟）

打开 `需求发现自动化脚本.py`，找到：

```python
headers = {
    'Authorization': 'Bearer YOUR_API_TOKEN_HERE',
    ...
}
```

替换 `YOUR_API_TOKEN_HERE` 为你的token

#### 步骤3：测试（1分钟）

```bash
python 需求发现自动化脚本.py
# 选择模式 3（完整模式）
```

✅ **完成标志**：能成功从Product Hunt获取数据

### Q3: 搜索很慢

**问题**：每个关键词需要等很久

**原因**：脚本加了延迟避免被封

**解决**：
- 减少关键词数量（5-8个即可）
- 使用快速模式（V2EX最快）
- 分批次搜索

## ⚠️ 限制说明

### 技术限制

| 限制 | 说明 | 解决方案 |
|------|------|---------|
| **反爬虫** | 知乎、Product Hunt有限制 | 使用Selenium或申请API |
| **请求频率** | 过快会被封IP | 脚本已加延迟 |
| **数据量** | 每次最多几百条 | 多次运行积累 |

### 法律限制

⚠️ **重要**：
- 仅用于个人学习和研究
- 不得用于商业爬虫
- 遵守网站服务条款
- 尊重robots.txt规则

### 使用建议

**推荐做法**：
1. ✅ 脚本用于快速发现（V2EX）
2. ✅ 手动深度分析（知乎、Product Hunt）
3. ✅ 结合Google Trends验证
4. ✅ 每天运行1-2次即可

**不推荐做法**：
1. ❌ 高频率运行（每小时）
2. ❌ 爬取大量数据（>1000条/天）
3. ❌ 不加延迟疯狂请求
4. ❌ 商业化使用爬取数据
